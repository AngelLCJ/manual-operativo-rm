¿De qué trata esta sección?

Esta sección describe los procesos que el sistema ejecuta de forma automática, sin que ninguna persona tenga que hacer algo para iniciarlos. Ocurren en segundo plano — a ciertas horas del día, cuando se cumple alguna condición, o de forma continua — para mantener la operación de las sucursales funcionando de manera ordenada y segura. El personal de la plataforma recibe avisos cuando alguno de estos procesos requiere su atención.

1

Contrato completado automáticamente

Sistema Consejera

¿Cómo funciona?

  • 🤖 El sistema revisa constantemente el avance de cada contrato activo.
  • 🔀 ¿Ya se atendieron todas las citas incluidas en el plan?
  • 🤖 Si todas las citas están marcadas como "atendida", el sistema cambia el estado del contrato a Completado.
  • 🤖 El sistema registra la fecha y hora exacta del cierre.
  • 🤖 Se envía una notificación automática a la consejera asignada para informarle que el plan de su paciente ha concluido.
  • ✅ El contrato queda cerrado. La consejera puede hacer seguimiento o proponer una renovación.
  • ❌ Si quedan citas pendientes o canceladas sin reprogramar, el sistema no cierra el contrato y la consejera recibe un aviso de "citas faltantes".
Nota operativa: Ninguna persona cierra el contrato manualmente. Si el sistema no lo cierra cuando debería, verificar que todas las citas del plan estén registradas como atendidas en la agenda.

2

Crear ruta clínica en primera cita de especialidad

Sistema Médica

¿Cómo funciona?

  • La paciente tiene su primera cita de una especialidad (por ejemplo, su primera consulta de dermatología).
  • 🤖 El sistema detecta que es la primera vez que esta paciente asiste a esa especialidad en el grupo.
  • 🔀 ¿Existe ya un expediente de seguimiento para esta paciente en esa especialidad?
  • 🤖 Si no existe, el sistema abre automáticamente un expediente de seguimiento para esa especialidad.
  • 🤖 El expediente queda vinculado al perfil de la paciente y disponible para la médica desde ese mismo momento.
  • ✅ La médica puede registrar notas, diagnósticos y próximos pasos directamente en el expediente recién creado.
  • ❌ Si la cita se cancela antes de iniciarse, el expediente no se crea — solo se genera cuando la cita se marca como "en curso" o "atendida".
Nota operativa: La médica no necesita crear el expediente manualmente. Estará disponible en la pantalla de la cita al momento de iniciar la consulta.

3

Reintento de notificación fallida (proceso nocturno)

Sistema

¿Cómo funciona?

  • 🤖 Cada noche, el sistema revisa si hay notificaciones que no se entregaron correctamente durante el día (por ejemplo, un WhatsApp que no llegó por problemas de conexión).
  • 🔀 ¿Hay notificaciones pendientes de entrega?
  • 🤖 Para cada notificación fallida, el sistema verifica cuántas veces ya se intentó enviar.
  • 🔀 ¿Se ha intentado menos de 3 veces?
  • 🤖 Si aún hay intentos disponibles, el sistema reenvía la notificación automáticamente.
  • ✅ Si el reenvío llega correctamente, la notificación se marca como "entregada".
  • ❌ Si después de 3 intentos sigue fallando, el sistema registra la notificación como "no entregada" y el equipo técnico recibe un reporte para revisarla manualmente.
Nota operativa: Este proceso ocurre automáticamente en la madrugada. Si una paciente reporta que no recibió un recordatorio, es posible que el sistema lo esté reintentando. Si persiste al día siguiente, avisar al soporte técnico.

4

Resumen de agenda — generación automática matutina

Sistema Recepcionista Gerente de Sucursal

¿Cómo funciona?

  • 🤖 Cada mañana, antes de que abra la sucursal, el sistema recopila todas las citas confirmadas del día para cada sucursal.
  • 🤖 Organiza las citas por horario, médica y tipo de servicio.
  • 🤖 Genera un resumen con el total de citas, los horarios de mayor demanda y cualquier cita que requiera atención especial (por ejemplo, primera consulta o paciente de plan).
  • 🤖 Envía el resumen automáticamente a la recepcionista y al gerente de cada sucursal, ya sea por correo o mediante una notificación en la plataforma.
  • ✅ El personal llega a la sucursal sabiendo exactamente qué esperar durante el día.
  • 🔀 Si el día no tiene citas agendadas, el sistema envía un aviso indicando que la agenda está vacía.
  • ❌ Si hay algún problema para generar el resumen, el sistema lo reintenta una vez más. Si falla dos veces, el equipo técnico recibe una alerta.
Nota operativa: El resumen llega antes de las 7:00 a.m. Si a las 8:00 a.m. no se ha recibido, revisar el correo no deseado o notificaciones de la plataforma y avisar a soporte si persiste.

5

Expiración automática de sesiones inactivas

Sistema Cualquier usuario

¿Cómo funciona?

  • Un usuario (recepcionista, médica, gerente, etc.) inicia sesión en la plataforma.
  • 🤖 El sistema comienza a monitorear el tiempo desde la última acción del usuario en la pantalla.
  • 🔀 ¿El usuario lleva más de 30 minutos sin tocar ninguna pantalla?
  • 🤖 Si es así, el sistema muestra un aviso: "Tu sesión está a punto de cerrarse por inactividad".
  • 🔀 ¿El usuario responde al aviso dentro de los próximos 2 minutos?
  • ✅ Si responde, la sesión continúa normalmente desde donde la dejó.
  • 🤖 Si no responde, el sistema cierra la sesión automáticamente y redirige a la pantalla de inicio de sesión.
  • ❌ Cualquier cambio no guardado se pierde al cerrarse la sesión. El sistema muestra un mensaje explicando por qué se cerró.
Nota operativa: Este proceso protege la información de las pacientes. Si compartes computadora con otra persona, asegúrate de cerrar sesión manualmente cuando termines, sin esperar a que expire sola.

6

Limpieza de horarios vencidos sin confirmar

Sistema Paciente

¿Cómo funciona?

  • Una paciente (o el bot de WhatsApp) reserva un horario de manera tentativa, pero no lo confirma de inmediato.
  • 🤖 El sistema bloquea ese horario y comienza a contar el tiempo establecido para confirmar (por ejemplo, 15 minutos).
  • 🔀 ¿La paciente confirmó la cita antes de que venza el tiempo?
  • ✅ Si confirmó, el horario queda reservado de forma definitiva y la paciente recibe su comprobante.
  • 🤖 Si el tiempo venció sin confirmación, el sistema libera el horario automáticamente.
  • 🤖 El horario vuelve a aparecer como disponible en la agenda para que cualquier otra paciente pueda agendarlo.
  • ❌ La paciente que no confirmó a tiempo no tiene ya ese horario garantizado y tendría que iniciar el proceso de agendar de nuevo.
Nota operativa: Este proceso evita que los horarios queden "bloqueados" por reservas abandonadas. Si una paciente llama diciendo que "ya lo había reservado", verificar en la agenda si el tiempo de confirmación venció.

Sincronización de datos entre módulos

Sistema Bot WA Gerente de Sucursal

¿Cómo funciona?

  • Un gerente o admin corporativo actualiza algún dato importante: por ejemplo, cambia el horario de atención de una médica o modifica los servicios disponibles en una sucursal.
  • 🤖 El sistema detecta el cambio en el módulo donde se realizó (por ejemplo, el módulo de configuración de sucursal).
  • 🤖 Automáticamente propaga el cambio a todos los módulos que dependen de esa información: la agenda, el bot de WhatsApp, el módulo de contratos y los reportes.
  • 🔀 ¿Algún módulo no pudo recibir el cambio correctamente?
  • ✅ Si todos los módulos se actualizan sin problema, el cambio entra en vigor de inmediato en toda la plataforma.
  • ❌ Si algún módulo no se actualizó correctamente, el sistema reintenta la sincronización. Si falla dos veces, el equipo técnico recibe una alerta y el módulo afectado muestra datos anteriores hasta que se resuelva.
Nota operativa: Si después de actualizar un horario o un servicio el bot de WhatsApp sigue mostrando información antigua, esperar unos minutos y probar de nuevo. Si persiste después de 10 minutos, reportar al soporte técnico.

Monitor de salud del sistema

Sistema Admin Corporativo

¿Cómo funciona?

  • 🤖 El sistema verifica continuamente que todos sus componentes funcionen correctamente: la plataforma web, el bot de WhatsApp, las notificaciones, la agenda y el acceso de usuarios.
  • 🔀 ¿Todos los componentes responden con normalidad?
  • ✅ Si todo está bien, el sistema continúa operando sin interrupciones y sin alertar a nadie.
  • 🔀 ¿Se detectó algún componente con problemas o lento?
  • 🤖 El sistema clasifica el problema por nivel de urgencia: bajo (lentitud), medio (función degradada) o alto (componente caído).
  • 🤖 Dependiendo de la urgencia, el sistema notifica al equipo técnico por correo o SMS con un resumen del problema detectado.
  • 🤖 Para problemas de urgencia baja o media, el sistema intenta resolver la situación automáticamente (reiniciando el componente afectado, por ejemplo).
  • ❌ Si el componente no se recupera, el admin corporativo y el equipo técnico reciben una alerta prioritaria para atenderlo de inmediato.
Nota operativa: El personal no necesita monitorear la salud del sistema — el sistema se vigila a sí mismo. Si experimentan lentitud o errores en la plataforma, pueden reportarlo al equipo técnico; es posible que ya estén trabajando en ello.
