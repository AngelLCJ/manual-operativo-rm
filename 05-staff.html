¿De qué trata esta sección?

Aquí se explica cómo registrar, configurar y gestionar a todo el personal que trabaja en las sucursales de Grupo Reina Madre: recepcionistas, consejeras, médicas, gerentes y administradores. Los procesos de esta sección los ejecutan el Platform Admin, el Admin Corporativo, los Gerentes de Sucursal y el área de Recursos Humanos (RRHH), cada uno con niveles de permiso distintos según su rol y con restricciones de jerarquía — nadie puede asignar permisos mayores a los que tiene.

1

Dar de alta un nuevo colaborador

Platform Admin RRHH Gerente de Sucursal Sistema

¿Cómo funciona?

  • El administrador autorizado (Platform Admin, RRHH o Gerente de Sucursal) accede al módulo Personal y Staff y hace clic en "Agregar colaborador".
  • Captura los datos del nuevo colaborador: nombre completo, correo electrónico institucional y teléfono.
  • Selecciona el rol que tendrá (Recepcionista, Consejera, Médica, etc.) y la o las sucursales donde trabajará.
  • 🔀 Restricción de jerarquía: ¿el rol seleccionado tiene un nivel igual o mayor al del administrador que da de alta?
  • ❌ Si el administrador intenta asignar un rol de nivel superior al suyo (por ejemplo, un Gerente creando otro Gerente), el sistema rechaza la operación. Un Gerente solo puede crear roles menores al suyo (Recepcionista, Consejera, Médica, Enfermería).
  • 🔀 ¿El correo ingresado ya está registrado en el sistema o en el directorio de identidad?
  • ❌ Si el correo ya existe en la base de datos o en el directorio de identidad, el sistema muestra un aviso y no permite duplicarlo.
  • 🔀 ¿Existen otros errores? (cédula profesional duplicada, servicios inválidos, turnos solapados entre sucursales)
  • ❌ Si hay datos inválidos el sistema los indica y bloquea el alta hasta corregirlos.
  • 🤖 Si todo es correcto, el sistema registra primero al colaborador en el directorio de identidad (Cognito) y después en la base de datos interna. El colaborador recibe automáticamente un correo de invitación de Cognito con una contraseña temporal para su primer ingreso.
  • 🔀 ¿El nuevo colaborador tiene un rol clínico (Médica, Especialista, Enfermería)?
  • Si tiene rol clínico, su perfil queda en estado "Pendiente de validación" y no aparece disponible en la agenda hasta que se completen sus documentos de onboarding.
  • ✅ Para roles no clínicos, el colaborador aparece en la lista del personal activo y puede iniciar sesión. Para roles clínicos, queda pendiente de validación de documentos.
Quién puede dar de alta y con qué alcance:
Platform Admin — puede crear staff con cualquier rol.
RRHH (corp_hr) — puede crear staff con roles hasta su nivel.
Gerente de Sucursal (clinic_admin) — puede crear staff solo para su sucursal y solo con roles inferiores al suyo (no puede crear otro Gerente ni Admin Corporativo).

Nota técnica: El correo del colaborador es el identificador en el directorio de identidad. Si algo falla en el paso de base de datos interna después de que el directorio ya creó la cuenta, el área de Sistemas puede resolverlo; el colaborador no podrá iniciar sesión hasta que se complete el proceso.

Asignar o cambiar el rol de un colaborador

Platform Admin RRHH Gerente de Sucursal Sistema

¿Cómo funciona?

  • El administrador autorizado (Platform Admin, RRHH o Gerente de Sucursal) busca al colaborador en la lista de personal y abre su perfil.
  • Hace clic en "Editar rol" y elige el nuevo rol de la lista disponible.
  • 🔀 ¿El nuevo rol tiene un nivel mayor al del administrador que hace el cambio?
  • ❌ No es posible asignar un rol de nivel superior al del propio administrador. El sistema bloquea el cambio.
  • 🔀 ¿El colaborador ya tiene un rol distinto al que se quiere asignar?
  • Si tiene rol previo diferente, el sistema muestra un panel con los permisos que perderá y los que ganará con el cambio. El administrador debe marcar una casilla de confirmación antes de poder guardar — el botón queda bloqueado hasta confirmar.
  • El administrador confirma el cambio.
  • 🤖 El sistema actualiza el rol en el directorio de identidad (Cognito) y en la base de datos interna. El cambio se sincroniza en ambos sistemas.
  • 🔀 ¿El nuevo rol es Admin de Plataforma (Platform Admin)?
  • 🤖 Si el nuevo rol es Platform Admin, el sistema activa automáticamente el segundo factor de autenticación (MFA). En su próximo inicio de sesión, el colaborador verá un código QR para configurar su app de autenticación (Google Authenticator o similar).
  • ✅ El perfil del colaborador muestra el rol actualizado. Si tiene una sesión activa en ese momento, el cambio se reflejará en su próximo inicio de sesión o en máximo 1 hora.
Quién puede cambiar roles: Platform Admin (cualquier rol), RRHH y Gerente de Sucursal (solo roles hasta su propio nivel de jerarquía). El Gerente solo puede cambiar el rol de personal de su propia sucursal.

Importante — Sesiones activas: El cambio de rol actualiza el directorio de identidad (Cognito) y la base de datos de inmediato. Sin embargo, si el colaborador tiene una sesión abierta, su acceso actual sigue siendo válido hasta que expire el token de seguridad — máximo 1 hora. No hay cierre de sesión automático al cambiar el rol. En la práctica esto significa que, si la médica o recepcionista está trabajando en ese momento, seguirá viendo sus permisos anteriores por un rato. Si hay urgencia, la solución es pedirle que cierre sesión y vuelva a entrar.

Si ocurre un fallo técnico durante el cambio, el sistema muestra un mensaje de error. El cambio puede demorar en reflejarse hasta el próximo inicio de sesión del colaborador.

Asignar o cambiar sucursal de un colaborador

Platform Admin Admin Corporativo RRHH Gerente de Sucursal

¿Cómo funciona?

  • Desde el perfil del colaborador, se accede a la sección "Sucursales asignadas".
  • 🔀 ¿Quién está haciendo el cambio?
  • Si es Gerente de Sucursal, solo puede agregar o quitar al colaborador de su propia sucursal. No puede asignarlo a sucursales de otros Gerentes ni moverlo entre sucursales.
  • Si es Admin Corporativo, RRHH o Platform Admin, puede agregar o quitar el colaborador de cualquier sucursal del grupo.
  • Se seleccionan o deseleccionan las sucursales deseadas y se guarda el cambio.
  • 🔀 ¿Quedaron cero sucursales asignadas después del cambio?
  • ❌ Si se eliminaron todas las sucursales del colaborador, el sistema lo desactiva automáticamente a nivel plataforma. Equivale a una baja lógica. Si esto no era la intención, se debe asignar al menos una sucursal o reactivar al colaborador manualmente.
  • 🤖 El sistema actualiza los accesos del colaborador. En la próxima sesión, verá únicamente la información de las sucursales que tenga asignadas.
  • ✅ El colaborador queda vinculado a las nuevas sucursales seleccionadas.
Nota sobre mover entre sucursales: Solo Admin Corporativo, RRHH y Platform Admin pueden trasladar a un colaborador de una sucursal a otra. El Gerente de Sucursal solo puede operar dentro de su propia sucursal.

Un colaborador puede estar en más de una sucursal simultáneamente (útil para médicas que rotan entre clínicas). El límite técnico del sistema es 13 sucursales por colaborador.

Configurar horarios de una médica

Gerente de Sucursal Admin Corporativo

¿Cómo funciona?

  • El Gerente de Sucursal o Admin Corporativo abre el perfil de la médica y va a la sección "Horarios de atención".
  • Selecciona la sucursal para la que quiere configurar el horario (una médica puede tener horarios distintos en cada sucursal).
  • Define los días de la semana en que atiende y el rango de horas de inicio y fin de consulta.
  • 🔀 ¿Hay traslapes con otro horario ya configurado para esa médica en la misma sucursal?
  • ❌ Si hay traslape de horarios, el sistema lo indica y no permite guardar hasta corregirlo. El sistema aplica un margen de 30 minutos de separación entre bloques.
  • Se confirman los horarios y se guardan.
  • 🤖 El sistema registra los bloques de turno de la médica. Estos turnos definen su plantilla de disponibilidad para la agenda.
  • ✅ Los turnos de la médica quedan configurados y la agenda puede utilizarlos para ofrecer citas.
Importante: Si se modifica el horario de una médica y ya hay citas agendadas en esos bloques, las citas existentes NO se cancelan automáticamente. Se debe revisar la agenda y notificar a las pacientes afectadas manualmente.

Ver perfil de un colaborador

Gerente de Sucursal Admin Corporativo

¿Cómo funciona?

  • Desde el módulo "Personal y Staff", se busca al colaborador por nombre o correo electrónico.
  • 🤖 El sistema muestra la lista filtrada con los colaboradores que coinciden con la búsqueda.
  • Se hace clic en el nombre del colaborador para abrir su perfil completo.
  • El perfil muestra: nombre, correo, teléfono, rol asignado, sucursales donde trabaja y sus horarios configurados.
  • 🔀 ¿Es un Gerente de Sucursal quien consulta el perfil?
  • Si es Gerente, solo puede ver perfiles de colaboradores asignados a su propia sucursal.
  • Si es Admin Corporativo o Platform Admin, puede ver el perfil de cualquier colaborador del grupo.
  • ✅ La información del colaborador se muestra completa y en tiempo real.

Editar datos de un colaborador

Gerente de Sucursal Admin Corporativo

¿Cómo funciona?

  • Desde el perfil del colaborador, se hace clic en el botón "Editar".
  • 🔀 ¿Qué dato se quiere modificar?
  • Nombre y teléfono: pueden ser editados tanto por el Gerente de Sucursal como por el Admin Corporativo.
  • Correo electrónico: el correo de un colaborador no puede cambiarse desde la plataforma. Es el identificador en el directorio de identidad (Cognito). Si se necesita cambiar el correo, es una operación técnica que debe solicitar a Sistemas directamente en Cognito.
  • Rol: solo lo puede cambiar un administrador autorizado (ver Escenario 2).
  • Se realizan los cambios y se hace clic en "Guardar".
  • 🤖 El sistema guarda los nuevos datos (nombre, teléfono) y los refleja de inmediato en el perfil.
  • ✅ Los datos actualizados quedan registrados.
Cambio de correo electrónico: Esta operación no está disponible en la interfaz de la plataforma. El correo es el identificador único del colaborador en el sistema de identidad. Para cambiar el correo de un colaborador, Sistemas debe realizar el cambio directamente en el directorio de identidad (Cognito) y actualizar también el registro en la base de datos. Contactar al área técnica para gestionarlo.

Baja de colaborador — flujo estándar (con aprobación de RRHH)

Gerente de Sucursal RRHH Sistema

¿Cómo funciona?

  • El Gerente de Sucursal abre el perfil del colaborador que ya no forma parte de la empresa y levanta una solicitud de baja.
  • 🤖 La solicitud queda en estado Solicitada y se notifica al área de RRHH o al administrador corporativo.
  • 🔀 RRHH o el administrador corporativo revisan la solicitud: ¿se aprueba la baja?
  • ❌ Si se rechaza, la solicitud se cierra y el colaborador sigue activo. El Gerente recibe la notificación del rechazo.
  • Si se aprueba, la solicitud pasa a estado Aprobada.
  • El ejecutor (RRHH, administrador corporativo u otro rol autorizado) ejecuta la baja formalmente.
  • 🤖 El sistema desactiva al colaborador en la base de datos (is_active = false) y en el directorio de identidad (Cognito). A partir de ese momento no puede hacer un nuevo inicio de sesión.
  • 🤖 Las citas futuras del colaborador se encolan automáticamente como pendientes de reasignación. La baja procede sin necesidad de esperar — las citas quedan marcadas para que el equipo las atienda después.
  • ✅ El colaborador queda en estado Inactivo. Sus registros, citas atendidas e historial se conservan íntegros.
Sobre las sesiones activas: Al ejecutar la baja, el colaborador no puede volver a iniciar sesión. Sin embargo, si tiene una sesión abierta en ese momento, seguirá activa hasta que expire el token de seguridad — máximo 1 hora. No hay cierre de sesión automático e inmediato. Si hay urgencia, contactar a Sistemas para una invalidación manual.

Los datos NO se borran. Los expedientes, citas y registros en los que participó el colaborador se conservan por integridad del historial médico y operativo.

Baja directa de colaborador (Platform Admin o Admin Corporativo)

Platform Admin Admin Corporativo Sistema

¿Cómo funciona?

  • El Platform Admin o Admin Corporativo abre el perfil del colaborador y hace clic en "Desactivar colaborador".
  • Confirma la acción en el cuadro de diálogo que aparece.
  • 🤖 El sistema desactiva al colaborador en la base de datos (is_active = false) y en el directorio de identidad (Cognito) en el mismo paso — sin necesidad de aprobación adicional.
  • 🤖 Las citas futuras del colaborador se encolan automáticamente como pendientes de reasignación. La baja procede sin necesidad de esperar — las citas quedan marcadas para que el equipo las atienda.
  • ✅ El colaborador queda en estado Inactivo. Sus registros, citas atendidas e historial se conservan íntegros.
Sobre las sesiones activas: El sistema bloquea la posibilidad de un nuevo inicio de sesión de forma inmediata. Sin embargo, si el colaborador tiene una sesión abierta en ese momento, seguirá activa hasta que expire el token de seguridad — máximo 1 hora. No hay cierre de sesión automático e inmediato. Si hay urgencia por razones de seguridad, contactar a Sistemas para invalidación manual.

Los datos NO se borran. Los expedientes, citas y registros en los que participó el colaborador se conservan por integridad del historial médico y operativo. Solo un administrador autorizado puede reactivar a un colaborador desactivado.

Admin Corporativo y Platform Admin — vista completa del personal

Platform Admin Admin Corporativo Sistema

¿Cómo funciona?

  • El Admin Corporativo o Platform Admin ingresa al módulo "Personal y Staff". A diferencia del Gerente de Sucursal, no se le redirige a una sucursal específica — ve el personal de todas las sucursales del grupo en una sola pantalla.
  • El Gerente de Sucursal, al acceder al mismo módulo, es redirigido automáticamente a la vista de su propia sucursal.
  • Admin Corporativo y Platform Admin pueden filtrar la lista por sucursal, por rol o por estatus (Activo / Inactivo) para encontrar rápidamente a quien buscan.
  • Desde esta vista centralizada, pueden ejecutar cualquier acción: dar de alta, editar, asignar roles, configurar horarios o desactivar a cualquier colaborador del grupo.
  • 🔀 ¿Se requiere gestionar solo una sucursal específica?
  • Pueden filtrar por sucursal para trabajar de forma enfocada, igual que haría un Gerente pero con alcance ampliado a todo el grupo.
  • ✅ Todas las acciones quedan registradas con la fecha, hora y el nombre del administrador que las realizó, para trazabilidad completa.
Diferencia clave entre roles:
Gerente de Sucursal — ve y gestiona únicamente el personal de su sucursal (redirigido automáticamente).
Admin Corporativo — ve y gestiona el personal de todas las sucursales del grupo desde una sola vista, sin filtros de clínica en las consultas.
Platform Admin — acceso irrestricto a todos los registros del sistema, incluyendo personal de todas las sucursales. Es el único rol que puede modificar a otro Platform Admin.
